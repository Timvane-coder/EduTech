Diagram 01 Unit Circle — 16 Standard Angles, Coordinates & Radians
How this diagram was built

Coordinate system

Centre placed at (250, 260) with a scale of 160 px = 1 unit, giving comfortable margins for labels. SVG y-axis is flipped vs maths convention, so upward angles subtract from the centre y: py = 260 − 160·sin(θ).

16 radial spokes

Each standard angle (every 30° and 45°) is drawn as a dashed <line> from the centre to the circle rim. Tip coordinates are pre-computed with exact trig values so labels remain true (e.g. 30° tip = (250+160·(√3/2), 260−160·(½))).

Coordinate labels

Each spoke tip carries two <text> nodes: the radian/degree measure (e.g. π/6 30°) and the exact (cos θ, sin θ) pair using Unicode fractions and radical symbols.

Reference triangle

Three lines form the right triangle at 60°: the spoke (hypotenuse), a horizontal leg to (330,260), and a vertical leg up to the rim point. A <rect> rotated to the corner marks the right angle.

Quadrant signs

Four small text labels in each quadrant remind students of the sign convention for sin and cos.
Diagram 02 Matrix Multiplication — Dot Product, Row × Column, Dimensions
How this diagram was built

Matrix brackets

True mathematical brackets are drawn as open-ended L-shapes using <path> elements — two short horizontals and a long vertical per bracket pair — which looks far more authentic than rectangles.

Colour-coded highlight bands

A light-blue <rect> with low opacity covers the active row of A and the active column of B. The highlighted result cell in C gets a stronger bordered rect to show exactly where those inputs converge.

Dashed connector arcs

Two curved <path> elements using cubic bezier control points trace from the highlighted row and column back to the result cell, with arrowhead polygons to show the flow of the dot product.

Breakdown annotation box

A dashed-border <rect> at the bottom hosts multi-line text spelling out C[0,0] = (1)(7)+(2)(9)+(3)(11) = 58, making the dot-product operation concrete.

Dimension annotations

Bracket-style dimension markers (three lines) alongside each matrix annotate m, k, and n so students can visually verify the inner-dimension-cancels rule.
Diagram 03 Linear Inequality — Feasible Region, Boundary Line, Test Point
How this diagram was built

Coordinate system

Origin at (210, 270), scale 50 px/unit. The boundary line 2x+y=6 has x-intercept (3,0) and y-intercept (0,6); both are computed exactly and mapped to pixel coordinates.

Shaded feasible region

A single <polygon> with semi-transparent blue fill covers the feasible half-plane. Its vertices are chosen to extend to the viewport edges while remaining on the correct side of the boundary line.

Boundary line

A single <line> drawn solid (not dashed) to indicate the ≤ inequality is inclusive — a key distinction from strict inequalities where a dashed line would be used.

Test point methodology

The origin is marked in green with an annotation showing the substitution 2(0)+0 = 0 ≤ 6 ✓, mirroring the standard textbook test-point method for identifying which side to shade.

Grid and intercept markers

Light <line> grid elements at each integer value provide reference. Filled <circle> dots mark the x-intercept and a note indicates the y-intercept lies above the viewport.
Diagram 04 Pythagorean Theorem — Squares, Right Triangle, 3-4-5 Example
How this diagram was built

Triangle construction

A 3-4-5 right triangle is placed with the right angle at (150, 350), scaled to 30 px/unit: leg a = 120 px (vertical), leg b = 90 px (horizontal).

Squares on each side

The axis-aligned squares on legs a and b are standard <rect> elements. The tilted square on the hypotenuse uses a <polygon> — its four vertices are computed by offsetting the hypotenuse endpoints by a perpendicular vector of length equal to the hypotenuse (150 px). The perpendicular unit vector of AB=(90,120) normalised to length 150 gives the offset (120,−90).

Colour coding

Each square uses a distinct semi-transparent fill (blue for a², green for b², amber for c²) so students visually associate each area with its corresponding leg at a glance.

Right-angle marker

A small <rect> at the corner vertex creates the standard square right-angle symbol without rotation, since the corner is already aligned to the axes.

Verification box

A dashed box in the lower right walks through the numeric check step-by-step: 9+16=25.
Diagram 05 Sine & Cosine Waves — Period, Amplitude, Phase, Zero Crossings
How this diagram was built

Coordinate system

Origin at (60, 170); x-scale 80 px = π/2 (so one full period spans 320 px to x=380); y-scale 100 px = 1 (amplitude 1 spans 100 px up/down from the axis).

Smooth sinusoidal curves

Each wave is drawn as a single cubic bezier <path> with control points placed at the quarter-period landmarks. The S-curve shape emerges naturally from the bezier tangents at these key points (zero crossings, peaks, troughs).

Phase difference visualisation

Sin (solid blue) starts at 0 and rises; cos (dashed red) starts at its peak 1. The same bezier shape is used for both, but shifted by one quarter-period (80 px) in x — directly encoding the phase relationship cos(x) = sin(x + π/2).

Annotations

Peak and trough points are marked with <circle> dots and labelled with exact coordinates. Zero-crossing dots are shown for sin. Bracket-style annotations on the left and bottom annotate amplitude (A=1) and period (T=2π).

Legend

A small legend box uses matching line styles (solid vs. dashed) to identify each curve — reproducing exactly the convention used in the plot itself.
Diagram 06 Quadratic Function — Vertex, Axis of Symmetry, Roots, Standard / Vertex / Factored Forms
How this diagram was built

Coordinate system

Origin at (180, 280), scale 60 px/unit. The function y = x²−4x+3 maps to SVG as py = 280 − 60·y_val.

Parabola curve

Seven key points are computed (at x = −1, 0, 1, 2, 3, 4) and joined with a smooth cubic bezier <path>. Control points are placed at intermediate x values so the curve hugs the exact mathematical shape — especially tight around the vertex (2, −1) which lies at the bottom of the parabola.

Three equivalent forms

A dashed info box lists all three representations of the same function: standard x²−4x+3, vertex (x−2)²−1, and factored (x−1)(x−3) — directly connecting the diagram's visual features (vertex at (2,−1), roots at 1 and 3) to their algebraic counterparts.

Axis of symmetry

A purple dashed vertical line at x=2 extends the full height of the diagram, visually confirming that the parabola is a perfect mirror image on either side.

Special points

Three distinct colours differentiate: vertex (purple), x-intercepts/roots (red), and y-intercept (amber) — matching standard textbook colour conventions.
